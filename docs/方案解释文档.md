# 伴学搭子 — 方案解释文档

## 一、项目概述

**伴学搭子** 是一款面向中小学生的学习陪伴 Web 应用，核心目标是：

- **陪伴**：通过虚拟角色对话，让孩子在独自学习时感到有人在身边
- **督促**：任务管理 + 番茄钟绑定，引导孩子主动规划和执行学习
- **乐趣**：角色人格化互动、彩蛋奖励、情绪安抚，让学习过程不枯燥

---

## 二、深层痛点剖析

### 痛点 1：孩子独自学习时缺乏陪伴感

家长不可能 24 小时陪读。孩子独自面对作业时容易走神、焦虑、拖延。市面上的学习工具（如番茄 TODO、Forest）都是冷冰冰的计时器，缺少情感温度。

**我们的解法**：引入 4 种关系人格（兄弟/闺蜜/情侣/学长），每种角色有独立的语气和话术风格，通过对话和点击互动模拟"有人在旁边"的感觉。

### 痛点 2：学习任务和专注行为是割裂的

大多数 TODO 应用只管列任务，番茄钟应用只管计时，两者之间没有关联。孩子不知道"我这 25 分钟到底在为哪个任务努力"。

**我们的解法**：每个任务有「专注」按钮，点击后自动绑定番茄钟并启动计时，页面顶部出现聚焦横幅，明确告诉孩子"我正在做什么、还剩多久"。完成后角色会在对话框里鼓励。

### 痛点 3：负面情绪无处释放

学习压力大时，孩子需要一个出口。但直接跟家长说"我不想写作业"往往得到的是批评，而不是理解。

**我们的解法**：对话框自动识别情绪关键词（烦、累、焦虑、不想…），触发安抚回复 + 系统实操建议（深呼吸法、任务拆解等），先共情再引导行动。

## 三、前人经验对比

| 产品 | 优点 | 不足 |
|------|------|------|
| **Forest（专注森林）** | 游戏化种树机制，有成就感 | 纯计时工具，无任务管理，无情感陪伴 |
| **番茄 TODO** | 任务+番茄钟结合 | 界面偏工具化，无角色互动，无情绪支持 |
| **Habitica** | RPG 化习惯养成 | 过于复杂，低龄用户上手难 |
| **CoStudy（线上自习室）** | 有社交氛围感 | 依赖真人在线，功能单一 |

**我们的差异化**：把"情感陪伴"和"学习管理"融合在一个页面里，不需要社交、不需要复杂规则，打开就能用。

## 四、三个备选方案及最终选择

### 方案 A：纯前端静态页面（HTML + LocalStorage）

- **优点**：零后端，部署最简单
- **缺点**：数据只在本机浏览器，换设备丢失；无法做统计持久化
- **放弃原因**：用户体验不完整，无法做历史数据查看

### 方案 B：Flask + SQLite 轻量全栈（最终选择）

- **优点**：
  - Python 技术栈简单，适合快速原型
  - SQLite 零配置，数据持久化在服务器
  - 同时用 LocalStorage 做离线降级
  - 前后端同一项目，部署一条命令
- **缺点**：SQLite 不适合高并发（但本项目不需要）
- **选择理由**：在开发效率、功能完整性、部署简易性之间取得最佳平衡

### 方案 C：前端 React/Vue + 后端 Node.js + 云数据库

- **优点**：技术更现代，可扩展性强
- **缺点**：开发周期长、构建配置复杂、依赖多
- **放弃原因**：对于 MVP 演示项目，过度工程化

## 五、技术实现说明

### 5.1 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Flask (Python) |
| 数据库 | SQLite |
| 前端 | 原生 HTML + CSS + JavaScript |
| 字体 | Google Fonts (Inter) |
| 头像生成 | DiceBear API (MIT 开源) |
| 音频 | Web Audio API 合成白噪音 |

### 5.2 API 接口一览

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/tasks | 获取全部任务（智能排序） |
| POST | /api/tasks | 创建任务 |
| PATCH | /api/tasks/:id | 编辑任务（名称/属性/状态） |
| DELETE | /api/tasks/:id | 删除任务 |
| POST | /api/tasks/from-memo | 备忘录一键转任务 |
| GET | /api/memos | 获取全部备忘 |
| POST | /api/memos | 创建备忘 |
| PATCH | /api/memos/:id | 编辑备忘 |
| DELETE | /api/memos/:id | 删除备忘 |
| GET | /api/settings | 读取用户偏好 |
| PATCH | /api/settings | 更新用户偏好 |
| GET | /api/stats/today | 今日统计 |
| GET | /api/stats/history?days=N | 历史统计 |
| POST | /api/stats/event | 上报番茄完成事件 |
| POST | /api/companion/reply | 伴学对话回复 |
| POST | /api/companion/tap | 点击角色轻互动 |

### 5.3 LocalStorage 持久化

用户偏好（关系人格、头像风格、番茄时长、夜间模式、音乐偏好）采用**双端持久化**策略：

- 每次修改同时写入后端 SQLite + 前端 LocalStorage
- 网络断开时，自动读取 LocalStorage 缓存，页面不白屏
- 恢复网络后，下次操作自动同步到服务器

**存储 key**：`bxtz_prefs_v1`，JSON 格式。

### 5.4 三个核心交互

**交互 1：任务绑定番茄钟（专注模式）**
- 点击任务的「专注」按钮 → 自动将任务状态改为"进行中" → 启动番茄钟倒计时
- 页面顶部出现橙色聚焦横幅，显示任务名 + 实时倒计时
- 支持暂停/继续/结束
- 专注完成 → 自动记入今日统计 → 角色在对话框发送鼓励

**交互 2：智能情绪对话**
- 用户在对话框输入任意文字
- 前端自动检测是否包含情绪关键词（烦、累、焦虑、不想、慌…）
- 普通消息 → 角色根据人格回复 1 条
- 情绪消息 → 角色安抚 + 系统实操建议（双重反馈）
- 完成类消息（搞定、做完…）→ 走鼓励话术池

**交互 3：番茄钟彩蛋系统**
- 每完成 3 的倍数个番茄 → 弹出角色专属彩蛋消息
- 4 种人格各有 2 条彩蛋文案，随机触发
- 进入休息时 → 角色提醒休息（人格化语气）

## 七、UI 设计理念

- **温暖不幼稚**：暖色调（#e86b3a 品牌橙），适合学生群体
- **简洁不简陋**：卡片式布局、柔和阴影、圆角设计
- **有反馈感**：按钮点击缩放、列表项滑入动画、番茄圆环呼吸光晕、聚焦横幅脉冲点
- **日/夜模式**：CSS 变量全局切换，一键适应不同学习时段

## 九、运行方式

```bash
# 1. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动
python run.py
# 访问 http://127.0.0.1:5001
```

## 十、部署说明

本项目部署在 **Railway** 平台（免费额度），流程如下：

1. 代码托管在 GitHub：`https://github.com/nick666777888/pa.git`
2. Railway 通过 GitHub 自动拉取代码并部署
3. `Procfile` 指定启动命令：`web: python3 run.py`
4. `run.py` 读取 Railway 注入的 `PORT` 环境变量，监听 `0.0.0.0`
5. Railway 自动安装 `requirements.txt` 中的 Python 依赖
6. 部署完成后自动分配公网域名

**线上地址**：https://web-production-89b26.up.railway.app/
